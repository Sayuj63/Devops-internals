Disaster Recovery

When the worst happens, we already know what to do.

This document defines the recovery tiers, the six rehearsed failure scenarios, and the post-mortem template. It is the source of truth that the on-call follows at 03:00 IST.

Failure scenarios

Each scenario lists detection, automatic response, manual playbook, recovery validation, and a post-mortem hook.

Detection signal — Prometheus alert PostgresUnreachable (no successful pg_up scrape for 30s) and CloudWatch alarm RDS-CPU-zero.

Automatic response — RDS Multi-AZ promotes the synchronous standby; writer DNS endpoint updates within ~45s. API pods retry with exponential backoff via pgbouncer.

Manual playbook

  1. Ack the alert in PagerDuty, post to #sim-ops-incidents.
  2. Run scripts/freeze.sh to block deploys for the duration.
  3. Confirm failover in RDS console — verify writer endpoint resolves to standby AZ.
  4. Tail API logs: kubectl logs -n sim-ops -l app=api -f | grep -i pool.
  5. If pods hold stale connections, rolling restart: kubectl rollout restart deploy/api -n sim-ops.
  6. Re-enable deploys, open incident review ticket.

Recovery validation — Synthetic check scripts/smoke-activate.sh issues one activation; verify 200 OK and a fresh row in audit_events.

Detection signal — All Route 53 health checks red; external synthetic from us-west-2 reports 100% error; AWS Health dashboard confirms regional impairment.

Manual playbook

  1. Declare a Sev-1 in PagerDuty; appoint Incident Commander.
  2. cd terraform/dr && terraform apply -auto-approve — promotes pilot-light to active.
  3. Promote RDS cross-region read replica: aws rds promote-read-replica --db-instance-identifier sim-ops-dr.
  4. Restore Vault: vault operator raft snapshot restore s3://sim-ops-dr/vault/latest.snap; unseal with Shamir keys (oncall + lead).
  5. Apply DR overlay: kubectl apply -k k8s/overlays/dr.
  6. Flip Route 53 weighted record to us-west-2 (100% weight).
  7. Warm caches; run scripts/smoke-dr.sh.
  8. Publish status-page update.

Recovery validation — Smoke test passes; Grafana DR dashboard receives metrics; sample activation appears in Kibana within 2 min.

Detection — vault_core_active gauge zero on all pods; new pod scheduling stalls because Vault Agent cannot issue tokens.

Manual playbook

  1. Inspect peers: vault operator raft list-peers.
  2. If majority lost, restore latest snapshot: vault operator raft snapshot restore -force latest.snap.
  3. Unseal with Shamir keys.
  4. Force-renew leases: kubectl rollout restart deploy/api deploy/worker -n sim-ops.

The EKS data plane keeps serving traffic when the control plane is degraded. Steps: stop the upgrade, cordon nodes, stand up a parallel cluster via the Terraform module (terraform/modules/eks), restore manifests via Kustomize, drain old node groups, then point the ALB target group at new node groups.

Trivy SBOM diff identifies the vulnerable layer. Apply k8s/base/isolation/netpol-deny-all.yaml to quarantine the workload, pin previous tag via kustomize edit set image api=ghcr.io/itm-sayuj/sim-ops/api:<previous>, revoke any Vault leases issued in the breach window with vault lease revoke -prefix database/creds/api.

The CertExpiry30d alert fires when any certificate is within 30 days of expiry. cert-manager renews automatically; if its renewal fails, force renewal with kubectl annotate certificate sim-ops-tls cert-manager.io/force-renew=true and confirm the rotated secret reaches the ingress.

Post-mortem template

Filed within 5 business days of any Sev-1 or Sev-2. The blameless template is checked into docs/postmortems/.

postmortem.md 
# Incident YYYY-MM-DD — <short title>

## Summary
One paragraph. What broke, who noticed, how long it lasted, customer impact.

## Timeline (IST)
- 13:04 — first alert fires
- 13:06 — oncall acks
- ...

## Root cause
Technical narrative. Cite logs, metrics, commit SHAs.

## Detection
Was the alert fast / accurate / actionable? If not, file a follow-up.

## Response
What worked. What did not. Who needed access they did not have.

## Action items
- [ ] Owner — concrete fix — due date
