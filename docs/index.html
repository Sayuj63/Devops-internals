The problem

Telecom provisioning is brittle, manual, and audit-hostile.

Operators ship millions of SIMs a day. The control plane that activates them is usually a tangle of spreadsheets, ticket queues, and shell scripts. We rebuilt that control plane the way an SRE would.

01 · activation latency

Hours, not seconds

Legacy provisioning rounds through CRM, billing, HLR, and OSS by hand. By the time a SIM is live, the customer has churned at the point-of-sale.

02 · drift

Every region is a snowflake

Ring deployments diverge because infra is clicked, not coded. Compliance changes can't ship safely across geographies.

03 · MTTR

Stuck SIMs go unseen

Without RED metrics and structured logs, mean time to recovery sits north of an hour. Customers feel every minute.

What we built

A real service, wrapped in a real DevOps lifecycle.

Not a slide deck. A FastAPI provisioning API, an async state-machine worker, a Postgres-backed inventory of 5,000 SIMs, and an operator dashboard — all delivered through a fully scripted DevOps lifecycle from Terraform to ELK.

pipeline {
  agent { label 'eks-builder' }
  options { timeout(time: 20, unit: 'MINUTES') }
  environment {
    REGISTRY = "ghcr.io/itm-sayuj/sim-ops"
    KUBECONFIG = credentials('kubeconfig-staging')
  }
  stages {
    stage('Lint')   { steps { sh 'ruff check app/' } }
    stage('Test')   { steps { sh 'pytest -q --cov=app' } }
    stage('Build')  { steps { sh "docker build -t $REGISTRY/api:$GIT_COMMIT -f docker/Dockerfile.api ." } }
    stage('Scan')   { steps { sh "trivy image --severity HIGH,CRITICAL $REGISTRY/api:$GIT_COMMIT" } }
    stage('Push')   { steps { sh "docker push $REGISTRY/api:$GIT_COMMIT" } }
    stage('Deploy') { steps { sh "kustomize build k8s/overlays/staging | kubectl apply -f -" } }
  }
}

Quality gates that fail the build

  • ruff + mypy --strict on the API package
  • pytest with 85% coverage floor on app/core
  • trivy HIGH/CRITICAL CVE gate on the runtime image
  • conftest OPA policies on Terraform plans (tags, encryption)
  • Cosign signature required before kubectl apply

Disaster Recovery

Six scenarios, rehearsed.

Each scenario carries a detection signal, an automatic response, a manual playbook, and a recovery validation step. The full game-day procedure lives in dr-plan.html.

Detection — CloudWatch DatabaseConnections drop to zero; Prometheus alert PostgresUnreachable fires within 30s.

Automatic response — RDS Multi-AZ promotes synchronous standby; CNAME flips; API pods reconnect via pgbouncer with exponential backoff.

Manual playbook

  1. Acknowledge the alert in PagerDuty; freeze deploys via scripts/freeze.sh.
  2. Confirm failover in RDS console; verify writer endpoint resolves to new AZ.
  3. Tail API logs for reconnection errors; restart any pods stuck on stale connections.
  4. Re-enable deploys; open incident review ticket.

Detection — Route 53 health checks fail across all ALB targets; external synthetic check from us-west-2 reports red.

Manual playbook

  1. Promote DR RDS cross-region read replica to standalone primary.
  2. Restore Vault from latest Raft snapshot in s3://sim-ops-dr/vault/.
  3. kubectl apply -k k8s/overlays/dr to spin up warm cluster.
  4. Flip Route 53 weighted record to us-west-2; warm CDN cache.
  5. Validate end-to-end with scripts/smoke-dr.sh.

Detection — vault_core_active gauge drops; VaultLeaderLost alert; pod secret renewals start failing.

Manual playbook

  1. Inspect vault status across pods; identify quorum loss vs. transient.
  2. If quorum lost: restore latest Raft snapshot via vault operator raft snapshot restore.
  3. Re-unseal with Shamir keys held by oncall + lead.
  4. Force-renew agent leases for API + worker pods.

Data plane keeps serving traffic. Cordon nodes, recreate the cluster from Terraform module, drain old nodes into the new managed node groups.

Trivy SBOM diff identifies the vulnerable image; NetworkPolicy isolates the affected workload; pin previous tag via kustomize edit set image; revoke any Vault leases issued in the breach window.

cert-manager renewal failure is caught by CertExpiry30d alert. Manual playbook: force renew via kubectl annotate certificate sim-ops cert-manager.io/force-renew=true and confirm new secret rotates into the ingress.
